# 胸有成影医疗影像数据管理系统 API 文档

## 1. 概述

本文档描述了胸有成影医疗影像数据管理系统的后端 API 接口，包括认证、用户管理、病例管理、影像管理、检测管理、模型管理、审计日志和个人中心等功能模块。

### 1.1 基础信息

- **接口基础路径**：`/api`
- **认证方式**：JWT Token
- **响应格式**：统一 JSON 格式
- **错误处理**：统一错误码和错误信息

### 1.2 响应格式

所有接口返回统一的 JSON 格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

- `code`：状态码，200 表示成功，其他表示失败
- `message`：返回消息
- `data`：返回数据

## 6. 检测管理接口

### 6.1 发起检测

**请求路径**：`/api/cases/{caseId}/detection`  
**请求方法**：POST  
**请求参数**：

```json
{
  "modelId": 1,
  "imageId": 1,
  "confidenceThreshold": 0.5
}
```

- `modelId`：模型ID（必填）
- `imageId`：图像ID（可选，不指定则检测病例的所有图像）
- `confidenceThreshold`：置信度阈值（可选，默认0.5）

### 11.3 检测任务一致性检查

**请求路径**：`/api/system/health/detection-consistency`  
**请求方法**：GET  
**请求头**：

- `Authorization: Bearer <token>`

**响应格式**：

```json
{
    "code": 200,
    "message": "success",
    "data": {
        "checkTime": "2026-04-24T10:00:00",
        "totalDetections": 100,
        "totalIssues": 2,
        "issues": [
            {
                "type": "ORPHAN_DETECTION",
                "detectionId": 15,
                "description": "检测任务关联的病例不存在"
            },
            {
                "type": "STUCK_PROCESSING",
                "detectionId": 23,
                "description": "检测任务卡在processing状态超过30分钟"
            }
        ]
    }
}
```

问题类型说明：
- `ORPHAN_DETECTION`：检测任务关联的病例不存在
- `INVALID_MODEL_REF`：检测任务关联的模型不存在
- `NO_FEEDBACK`：已完成检测任务无反馈记录
- `STUCK_PROCESSING`：检测任务卡在 processing 状态超过30分钟
- `FAILED_NO_ERROR`：失败检测任务无错误信息
